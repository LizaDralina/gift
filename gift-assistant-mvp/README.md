##  Gift Assistant — Помощник для выбора подарка

**Прозрачный, быстрый и персонализированный сервис для выбора подарков** с использованием гибридной системы рекомендаций (правила + ML) и импортом интересов из VK.

---

##  Описание проекта

Gift Assistant — это веб-приложение, которое помогает подобрать подарок для конкретного человека с учётом:

- бюджета
- возраста и пола
- повода (День рождения, Новый год и другие)
- типа отношений (друг, партнёр, коллега, родственник)
- интересов получателя (ручной ввод или импорт из VK)
- исключений (что дарить НЕ нужно)

### Ключевые особенности

- **Rule-based фильтрация** — строгое соблюдение бюджета, возраста и исключений. Ни один товар, нарушающий обязательные ограничения, не попадает в выдачу.

- **ML-ранжирование** — градиентный бустинг (GradientBoostingRegressor из scikit-learn) для сортировки наиболее релевантных подарков. Используется гибридная формула: 70% эвристики + 30% ML.

- **Прозрачные объяснения** — к каждому подарку прилагается список причин рекомендации (например: «Соответствует интересу "космос"», «Попадает в бюджет», «Подходит для Дня рождения»).

- **Импорт интересов из VK** — по публичной ссылке на профиль без авторизации получателя. Система извлекает интересы, музыку, книги, игры, группы и маппит их на внутренний словарь.

- **Готовность к развёртыванию** — Docker Compose для запуска одной командой.

##  Архитектура проекта

Проект реализован как классическое клиент-серверное веб-приложение.

### Основные компоненты

**Frontend (клиентская часть)**
- Назначение: пользовательский интерфейс, формы ввода данных, отображение карточек рекомендаций с объяснениями

**Backend API (серверная часть)**
- Назначение: бизнес-логика, обработка HTTP-запросов, аутентификация, фильтрация и ранжирование товаров

**ML-подсистема**
- Технологии: scikit-learn (GradientBoostingRegressor), joblib, numpy
- Назначение: загрузка обученной модели, формирование вектора признаков, расчёт ML-оценки релевантности

**Сервис интеграции с VK**
- Технологии: requests, VK API (users.get, groups.get)
- Назначение: импорт публичных интересов из профиля VK по ссылке

**База данных**
- Технологии: SQLite (прототип), SQLAlchemy ORM
- Назначение: хранение пользователей, получателей, товаров, интересов

**Инфраструктура**
- Технологии: Docker, Docker Compose
- Назначение: контейнеризация и унифицированный запуск всех компонентов

### Паттерны проектирования

- **REST API** — слабосвязанное взаимодействие клиента и сервера через HTTP-методы (GET, POST, PATCH, DELETE) с передачей данных в JSON.

- **MVC + Service Layer** — разделение ответственности: модели (ORM), контроллеры (routers), бизнес-логика (services), инфраструктура (core).

- **Pipeline (конвейер)** — последовательная обработка запроса рекомендаций: фильтрация по правилам → ML-ранжирование → формирование объяснений.

##  Инструкция по запуску

### Шаги для запуска

Перейдите в папку проекта:

cd gift-assistant-mvp

запустите :
docker compose up --build

открыть сайл http://localhost:5173/dashboard , зарегестрироваться , загрузить каталог подарков products.csv , и заполнить данные о получателе, можно импортировать из его вк интересы
